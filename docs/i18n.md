# Internationalisation

The application ships with three locales: **Portuguese** (reference),
**English** and **Spanish**. All UI strings flow through a single
`t(key, **kwargs)` helper.

## Layout

```
src/i18n/
├── translations.py        # Loader + AVAILABLE_LANGUAGES
└── locales/
    ├── pt.json            # Reference (every key must exist here)
    ├── en.json
    └── es.json
```

The reference language is set in
`src/i18n/translations.py::DEFAULT_LANGUAGE` (`"pt"`). When a key is
missing in the active locale, the loader falls back to `pt.json`; if it
is missing in `pt.json`, the bare key path is rendered.

## Using `t(...)` from a page

```python
from src.i18n import t

st.subheader(t("eda.title"))
st.warning(t("upload.schema.required_missing", cols=", ".join(missing)))
```

Placeholders use Python's `str.format` syntax (`{var}`).

## Adding a new key

1. Add the key to `src/i18n/locales/pt.json`. Group keys by feature
   prefix (`eda.`, `pipeline.`, `spatial.`, `comparative.`, ...).
2. Mirror the key in `en.json` and `es.json` with the translated text.
3. Run the audit:
   ```bash
   python -m scripts.i18n_audit
   ```
4. The audit prints `pt.json` keys missing from `en`/`es` — the build
   should report **none** in both directions.

## Adding a new language

1. Create `src/i18n/locales/<code>.json` with the same keys as
   `pt.json`. Start by copying `pt.json` and translating in place; the
   audit script will tell you what is missing.
2. Register the code in
   `src/i18n/translations.py::AVAILABLE_LANGUAGES`:
   ```python
   AVAILABLE_LANGUAGES: dict[str, str] = {
       "pt": "Portugues",
       "en": "English",
       "es": "Espanol",
       "fr": "Francais",   # new
   }
   ```
3. Restart Streamlit. The sidebar selector picks the new entry up
   automatically.

## Dynamic strings (avoiding hard-coded user-visible text)

Any string that the user can read must come from a locale file — even
short ones like `"(none)"` (which is `common.none`). Constants in
`src/config/settings.py` such as `REGRESSION_PRESETS` reference *i18n
keys*, not literal labels, and the page renders the label by calling
`t(...)` at render time.

## Reloading translations during development

Streamlit hot-reloads source files but does **not** automatically
reload JSON imports. After editing a locale, either:

```python
# In a Python REPL with the project on the path
from src.i18n.translations import reload_translations
reload_translations()
```

…or simply restart `streamlit run`.

## Style conventions

- **Sentence case** for tab labels and section headings
  (`"Pipeline e Processamento"`, not `"Pipeline E Processamento"`).
- **Imperative mood** for buttons (`"Aplicar pipeline"`, `"Carregar
  Dataset"`).
- Diacritics: keep them (`"Análise Espacial"`); avoid ASCII-only
  fallbacks unless strictly necessary.
- Technical terms (`Q10`, `LISA`, `STL`, `Moran's I`) are kept in the
  original form across all locales.
